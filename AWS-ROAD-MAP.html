🔧 Troubleshooting Common Fixes
🚫 Redis not found

Fix: sudo systemctl start redis-server. Falls back to MemoryStore.

🔴 Prisma migration fail

Fix: npx prisma db push or npx prisma migrate resolve --applied

🔄 PM2 crashed

Fix: pm2 logs finflow-backend --lines 100. Common: missing env, DB down, port in use.

🌐 502 Bad Gateway

Fix: pm2 list → pm2 restart finflow-backend. Check sudo nginx -t.

🔑 JWT_SECRET missing

Fix: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))". Add to .env. PM2 restart.

📦 S3 upload fails

Fix: Check AWS creds in .env. Verify bucket & region. Test: aws s3 ls s3://finflow-assets

❓ FAQ Q&A
Q: How to add a new module?

A: 1. Update prisma/schema.prisma → migrate. 2. Create Repository. 3. Create module (routes/controllers/services/validators). 4. Register in src/app.js.

Q: Why separate Repositories from Services?

A: Separation of Concerns. Services decide WHAT to do, Repositories handle HOW to fetch/save. Easier testing + modularity.

Q: Why 1-hour S3 presigned URLs?

A: Security — if leaked, URL expires in 60 minutes. Protects sensitive documents and videos.

Q: How does refresh token rotation work?

A: Each refresh issues a new token & revokes the old one. Theft detection: if a compromised token is used after rotation, all sessions are revoked.

Q: Do rate limits reset on server restart?

A: With Redis → persist. With MemoryStore → reset. Production uses Redis, so limits survive restarts.
