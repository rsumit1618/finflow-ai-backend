🏗️ Modular Architecture & API Flow Standard

The backend is built using a Modular Layered Architecture to ensure independent scaling of features like Auth, Expenses, and Document AI.

📌 Request Execution Journey
// 1. Client sends request to Route
Route (URL definition & Swagger JSDoc)
     ↓
Middleware (Auth Check, Rate Limiting, Zod Validation)
     ↓
Controller (Handle HTTP req/res, calling services)
     ↓
Service (Business logic, domain rules, calling repositories)
     ↓
Repository (Prisma database access ONLY)
     ↓
PostgreSQL (Data persistence)
📁 Phase 5 — Document AI & Storage New
✅ S3 Hierarchy (AI-Ready)

Files are organized into logical folders for future Vector search/RAG processing.

  • documents/{userId}/pdf/ — Financial docs
  • documents/{userId}/image/ — Receipts/Photos
  • documents/{userId}/other/ — General files
✅ Secure Document Access
  • Private Bucket: No public access allowed.
  • Presigned URLs: Temporary 1-hour access links.
  • User Isolation: Strict DB check for document ownership.

❓ Technical Q&A FAQ
Q: How do I add a new feature (Module)?

A: 1. Update schema.prisma & Migrate. 2. Create Repository. 3. Build Module folder (routes/controllers/services). 4. Register in app.js.

Q: Why use Repositories separately from Services?

A: Separation of Concerns. Services handle business logic (deciding WHAT to do), while Repositories handle database implementation (HOW to save/fetch).

Q: Why do S3 URLs expire in 1 hour?

A: For security. If a URL is leaked, it becomes useless after 1 hour. This protects sensitive financial documents from unauthorized long-term access.
